# 开发变更记录 (CHANGELOG_DEV.md)

## v1.07 (2026-05-13) - 细节打磨与图片对齐微调版

### 1. UI 细节优化：图片垂直对齐二次微调
- **调整内容**：将 `.q-img` 的 `vertical-align` 值从 `-0.25em` 精确修正为 `-0.18em`。
- **原因**：根据用户反馈，之前的调整导致图片位置在某些浏览器下略微偏低。
- **效果**：图片现在会稍微向上移动约 1-2 像素，使其中心点更完美地落在文字行高（Line-height）的中轴线上。

### 2. 交互修正：导出文件名规范化
- **调整内容**：将 `exportData` 函数中的下载文件名从 `ai_trainer_backup.json` 改为 `social_work_sim_backup.json`。
- **原因**：原有文件名带有旧项目痕迹，不符合当前“社会工作模拟系统”的定位。

## v1.06 (2026-05-13) - 统计联动与重练进度同步版

### 1. 逻辑重构：会话级统计同步
- **调整内容**：
    - 重写了 `getAnsweredCount` 和 `getCorrectWrongCounts` 函数。
    - 当 `practiceMode` 切换为 `wrong` 或 `fav` 时，统计数据将自动从 `sessionAnsweredMap`（临时会话图）中读取，而非 `answeredMap`（全局持久化图）。
    - 统计范围被限定在 `activeIds` 内，确保左侧面板显示的“正确/错误数”与当前练习的题目总数一一对应。
- **原因**：解决用户在“错题重练”模式下，左侧统计面板依然显示全库历史数据（如正确数 4，错误数 9），导致进度条和正确率百分比逻辑混乱的问题。
- **背景**：实现了 UI 状态与答题模式的深度同步，让用户在重练时能清晰看到“当前这 6 道错题我答对了多少”。

## v1.05 (2026-05-13) - 重练模式增强与图文排版修复版

### 1. 功能增强：独立重练 session 系统
- **调整内容**：在 `js/app.js` 中引入了 `sessionAnsweredMap`。当用户从错题本或收藏夹启动“开始重练”时，系统会自动清空并使用该临时进度图。
- **原因**：解决用户在重练模式下因题目已有全局答题记录而无法重新作答的问题。
- **背景**：实现了“重练即刷新”的逻辑，用户可以在不干扰全局正确率统计的前提下，对薄弱环节进行多次反复测试。

### 2. UI 修复：题目图文流式排版重构
- **调整内容**：
    - **布局降级优化**：将 `.q-title` 的 CSS 布局从 `flex` 降级回标准的 `display: block`。
    - **消除意外换行**：移除了 Flex 容器对纯文本节点与图片元素之间的强制换行干扰，解决了图片单独占行、文字强行错位的 Bug。
    - **精确垂直对齐**：通过 `vertical-align: -0.25em` 替代原有的 `sub` 方案，实现了图片中心与文字行中心的像素级重合。
- **原因**：解决用户反馈的“图片高于文字”以及“图片文字换行”等排版瑕疵。

## v1.04 (2026-05-13) - 稳定性修复与主题兼容优化

### 1. 关键稳定性修复：错题本/收藏夹导航崩溃
- **调整内容**：
    - 移除了 `js/app.js` 中 `views` 对象对已废弃 `view-handson` ID 的引用。
    - 在 `switchView` 函数中添加了安全检查，确保切换时视图对象存在且不为 null。
- **原因**：由于 `view-handson` 在 v1.01 中被移除但 JS 引用未清理，导致 `Object.values(views).forEach` 执行时抛出 null 指针异常，阻断了后续的视图切换逻辑，表现为“错题本打不开”。

### 2. 主题兼容性增强：自动模式视觉修复
- **调整内容**：
    - 为 `global-sticky-header`（答题模式行）新增了 `box-shadow: 0 1px 0 var(--border-color)`，以增强边框在不同主题切换时的对比度。
    - 优化了 `renderList` 函数，在渲染错题和收藏列表时添加了 ID 类型强制转换（`Number(q.id)`）和数据结构校验。
- **原因**：解决用户反馈的“自动模式下边框消失”以及“收藏夹打开空白”的问题。
- **背景**：确保在系统默认主题（Auto）下，CSS 变量能够稳定解析并提供足够的视觉区分。

## v1.03 (2026-05-13) - 界面精致化与交互增强版

### 1. 交互增强：动态激励语系统
- **调整内容**：将左侧边栏顶部的静态激励语“勤学苦练，必有所获”替换为动态轮询系统。
- **原因**：用户需要多样的激励内容以缓解备考压力。
- **背景**：新增了包含 14 条经典激励语的库，每次页面刷新或重新打开时随机展示，增强系统的人性化体验。

### 2. UI 深度对齐修复
- **调整内容**：
    - **消除背景冲突**：统一了题目区域与解析区域的背景色为 `var(--surface-color)`，解决了侧边栏颜色不一致导致的视觉割裂感。
    - **线条精准对齐**：重构了 `.glass-card` 和 `.card-body` 的内边距与边框逻辑。移除了可能导致像素级错位的负 Margin 方案，改用容器级 Padding 控制。
    - **吸顶栏对齐**：微调了顶部控件栏（global-sticky-header）的 Margin 和 Padding，使其与下方的卡片和左侧边栏在视觉上完美对齐。
- **原因**：解决用户反馈的线条错位、背景色冲突等视觉瑕疵，提升系统精致感。

### 3. 视觉优化：图片文字流式对齐
- **调整内容**：将题目中插入图片的 `vertical-align` 从 `middle` 优化为 `sub`，并微调高度至 `1.25em`。
- **原因**：解决图片在文字行中产生的视觉下坠或上漂问题。
- **背景**：确保题号、文字和图片在同一基准线上，实现更自然的阅读流。

## v1.02 (2026-05-12) - 深度联动与功能恢复版

### 1. 核心逻辑重构：全动态联动筛选
- **调整内容**：重写了左侧题号面板（Sidebar Grid）的渲染逻辑。
- **原因**：原本的列表是静态生成的，切换章节或题型筛选后，列表内容不随之变化，且进度统计与当前显示内容不匹配。
- **背景**：用户需要快速定位特定章节的题目，动态筛选能让练习更具针对性（如：选择第一章后，左侧只显示第一章的 176 道题，统计也只针对这 176 题）。

### 2. 功能恢复：进度备份管理
- **调整内容**：在顶部控制栏右侧重新设计并上线了“导入进度”、“导出进度”和“重置数据”按钮。
- **原因**：在 v1.01 简化 UI 时误删/隐藏了备份入口，导致用户无法保存或迁移做题进度。
- **背景**：单机版系统依赖浏览器缓存，进度备份是防止数据丢失、支持跨设备学习的核心保障。

### 3. 视觉与排版二次优化
- **调整内容**：
    - 将题目图片（.q-img）对齐方式改为 `align-items: baseline`。
    - 图片高度限制在 `1.2em`。
    - 移除了题目与图片之间的所有多余换行符和空格。
- **原因**：解决题号、图片、文字“三行占位”问题。
- **背景**：实现极致的“单行流式显示”，让用户在阅读长题目时视觉更连贯。

## v1.01 (2026-05-12) - 社会工作者考试定制版

### 1. 架构调整：从 AIT3 迁移并简化
- **调整内容**：移除了“理论练习”与“案例实操”的双模式切换，统一为单一练习流。
- **原因**：用户提供的 MD 题目目前均为单选题和多选题，没有主观案例实操题。
- **背景**：为了使界面更专注于当前的“章节练习”和“历年真题”，减少冗余按钮。

### 2. 实时同步方案：新增 run_simulator.py
- **调整内容**：引入了一个轻量级 Python HTTP 服务器，在页面刷新时自动触发 `build_data.py`。
- **原因**：MD 转换 JS 脚本的手动执行成本太高。
- **背景**：用户需要频繁修改 MD 文档并即时在网页上查看效果，实现“所见即所得”。

### 3. 正确率逻辑修正
- **调整内容**：将正确率计算公式改为 `答对总数 / 全库总题数`，且进度从 0% 开始增长。
- **原因**：原逻辑是基于已答题目的扣减，无法体现全库的整体复习进度。
- **背景**：符合备考场景下“刷完一整本书”的进度心理。
