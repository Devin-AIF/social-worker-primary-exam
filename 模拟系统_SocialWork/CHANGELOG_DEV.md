# 开发变更记录 (CHANGELOG_DEV.md)

## v1.02 (2026-05-12) - 深度联动与功能恢复版

### 1. 核心逻辑重构：全动态联动筛选
- **调整内容**：重写了左侧题号面板（Sidebar Grid）的渲染逻辑。
- **原因**：原本的列表是静态生成的，切换章节或题型筛选后，列表内容不随之变化，且进度统计与当前显示内容不匹配。
- **背景**：用户需要快速定位特定章节的题目，动态筛选能让练习更具针对性（如：选择第一章后，左侧只显示第一章的 176 道题，统计也只针对这 176 题）。

### 2. 功能恢复：进度备份管理
- **调整内容**：在顶部控制栏右侧重新设计并上线了“导入进度”、“导出进度”和“重置数据”按钮。
- **原因**：在 v1.01 简化 UI 时误删/隐藏了备份入口，导致用户无法保存或迁移做题进度。
- **背景**：单机版系统依赖浏览器缓存，进度备份是防止数据丢失、支持跨设备学习的核心保障。

### 3. 视觉与排版二次优化
- **调整内容**：
    - 将题目图片（.q-img）对齐方式改为 `align-items: baseline`。
    - 图片高度限制在 `1.2em`。
    - 移除了题目与图片之间的所有多余换行符和空格。
- **原因**：解决题号、图片、文字“三行占位”问题。
- **背景**：实现极致的“单行流式显示”，让用户在阅读长题目时视觉更连贯。

## v1.01 (2026-05-12) - 社会工作者考试定制版

### 1. 架构调整：从 AIT3 迁移并简化
- **调整内容**：移除了“理论练习”与“案例实操”的双模式切换，统一为单一练习流。
- **原因**：用户提供的 MD 题目目前均为单选题和多选题，没有主观案例实操题。
- **背景**：为了使界面更专注于当前的“章节练习”和“历年真题”，减少冗余按钮。

### 2. 实时同步方案：新增 run_simulator.py
- **调整内容**：引入了一个轻量级 Python HTTP 服务器，在页面刷新时自动触发 `build_data.py`。
- **原因**：MD 转换 JS 脚本的手动执行成本太高。
- **背景**：用户需要频繁修改 MD 文档并即时在网页上查看效果，实现“所见即所得”。

### 3. 正确率逻辑修正
- **调整内容**：将正确率计算公式改为 `答对总数 / 全库总题数`，且进度从 0% 开始增长。
- **原因**：原逻辑是基于已答题目的扣减，无法体现全库的整体复习进度。
- **背景**：符合备考场景下“刷完一整本书”的进度心理。
