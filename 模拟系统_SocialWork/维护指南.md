## 3. 核心维护规则

### 3.1 理论模块

- `mock` 模式必须从 `questionsData` 全量池抽题，不能被错题、收藏或局部练习池污染。
- `random` 模式若定义为“未答优先”，则必须明确排除 `answeredMap` 中已存在记录的题目。
- `getAnsweredCount`、`finishExam`、进度条展示三者必须保持同一统计口径。
- 交卷后若希望题目被系统识别为“已完成”，必须回写 `answeredMap`，否则会出现随机题池与统计失真。
- 导入备份时必须清洗 `wrongList`、`favorites`、`answeredMap` 的 ID 类型和异常值。
- 题目缺失时必须展示错误提示，不允许因为单题异常导致整页脚本中断。

### 3.2 实操模块

- 所有实操初始化逻辑应集中在 `DOMContentLoaded` 之后执行，避免全局直接操作 DOM。
- 实操题号导航应支持直接定位，不应保留误导性的平滑移动轨迹。
- 侧栏高亮追踪必须防抖，避免因 `IntersectionObserver` 波动导致题号横跳。
- 展开答案、核对填空、切换套卷时需要考虑页面高度突变带来的定位漂移。
- 套卷切换、理论/实操切换后，优先恢复到用户上次停留的小题位置。
- `type: "text"`、`type: "upload"`、`type: "text_upload"` 三类渲染分支必须都保留，不能因重构丢失。

### 3.3 代码与渲染安全

- 代码复制逻辑必须输出纯净文本，不能把 HTML 输入框或富文本结构一并复制出去。
- Markdown 渲染器必须保留对疑似 Base64 大文本的拦截。
- 数据预览和答案展开不能破坏当前焦点与滚动体验。

## 4. 当前稳定实现摘要

### 已完成的关键修复

- 随机出题改为未答优先。
- 模拟组卷改为未答优先，不足时自动从旧题回补。
- 进度显示支持区分“全局进度”和“当前练习池进度”。
- 底部正确/错误统计统一按全库已答结果统计。
- 交卷结果会回写到 `answeredMap`。
- 实操侧栏改为“位置记忆 + 金轴判定 + 临时锁定”。
- 实操点击题号改为直接跳转，不再保留平滑痕迹。
- 理论题号高亮改为内阴影，避免蓝框溢出边界。

- [ ] 展开“核对填空与查看解答”后，高亮不会乱跳到别的章节。
- [ ] 在实操一和实操二之间来回切换，能恢复上次停留位置。

- 为 `updateActiveIds`、`getAnsweredCount`、`finishExam`、`importData` 增加最小自动化测试。
